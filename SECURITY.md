# SECURITY

Responsible disclosure

If you discover a security vulnerability in this project, please report it privately to the repository owner via email (add your contact) or GitHub private issue. Avoid posting vulnerabilities publicly until they've been addressed.

Suggested process

- Include steps to reproduce, affected versions, and an assessment of severity.
- Provide a suggested fix if possible.
- Expect a timely response; maintainers should acknowledge within 72 hours and aim to resolve or mitigate quickly.

Public disclosures

After a vulnerability is fixed, maintainers may publish a CVE or advisory if appropriate.

Notes
